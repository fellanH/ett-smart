  <name>Task 2: Implement CSV parsing with column auto-detection</name>
  <files>webapp/app.py</files>
  <action>
Extend app.py to handle uploaded CSV:

1. When file is uploaded, read with pandas:

   ```python
   df = pd.read_csv(uploaded_file)
   st.session_state.uploaded_df = df
   ```

2. Auto-detect columns by looking for common patterns:
   - Company name: columns containing "company", "företag", "namn", "name" (case-insensitive)
   - Org number: columns containing "org", "orgnr", "organization", "nummer" (case-insensitive)

3. Display detected mapping with `st.selectbox` for each field:
   - "Company Name Column" - dropdown of all columns, pre-selected to detected
   - "Organization Number Column" (optional) - dropdown with "None" option

4. Store selected columns in session_state:

   ```python
   st.session_state.company_col = selected_company_col
   st.session_state.org_col = selected_org_col
   ```

5. Show preview of first 5 rows using `st.dataframe(df.head())` after mapping confirmed

6. Add "Confirm Mapping" button that stores mapping and advances workflow state
   </action>
   <verify>Upload a test CSV with columns "Company Name" and "Org Number" - both should be auto-detected. Changing selection updates session state.</verify>
   <done>Columns auto-detected correctly, user can override via dropdowns, mapping stored in session_state</done>
   </task>

<task type="auto">
  <name>Task 3: Add input validation with user-friendly errors</name>
  <files>webapp/app.py</files>
  <action>
Add validation layer after CSV upload:

1. File format validation:
   - Check file extension is .csv (already enforced by uploader, but double-check)
   - Try pd.read_csv, catch exceptions and show `st.error("Invalid CSV format: {error}")`

2. Required columns validation:
   - After mapping, verify company name column is selected (not None)
   - Show `st.warning("Please select the company name column")` if missing

3. Data validation:
   - Check for empty company names: `df[company_col].isna().sum()`
   - Check for duplicates: `df[company_col].duplicated().sum()`
   - Show `st.info(f"Found {n} empty rows and {m} duplicates - these will be skipped")`

4. Batch size validation:
   - Count valid rows: `valid_count = len(df) - empty - duplicates`
   - If valid_count > 50: `st.warning("Large batch detected ({n} companies). Processing may take several minutes.")`
   - If valid_count == 0: `st.error("No valid companies found in the file")`

5. Add workflow state management:
   ```python
   st.session_state.workflow_step = "upload" | "mapping" | "ready_to_process"
   ```
   Only show "Start Processing" button when workflow_step == "ready_to_process"
   </action>
   <verify>Upload CSV with missing company column - shows warning. Upload CSV with duplicates - shows info message. Upload empty CSV - shows error.</verify>
   <done>All validation cases handled with appropriate user feedback, workflow advances correctly</done>
   </task>
